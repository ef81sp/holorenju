# Bug: 四三を見逃して禁手追い込みに回る

## 概要

CPUが四三で即勝ちできる局面で四三を打たず、より長い禁手追い込み(VCF-forbidden)ルートを選択してしまう。

## 再現対局

- 分析ファイル: `analysis-2026-02-14T12-23-56-039Z.json`
- フィルタ: `--tag=vcf-forbidden --matchup=hard --moves=20-40`
- 対局 #164: hard vs hard, B wins (forbidden, 40手, vcf-6)

## 症状

- 禁手追い込みシーケンスの前に、H4 への着手が四三になっている
- 四三はそのまま勝ちなので、禁手追い込みに回る必要がない
- CPUが四三勝ちよりも禁手追い込みを優先してしまっている

## 想定される原因

- 評価関数またはムーブオーダリングで、四三勝ちのスコアが禁手追い込みより低く評価されている可能性
- または探索で四三勝ちの手が候補として見つかっているのに、禁手追い込みルートの方が先に発見・選択されている可能性

## 確認方法

```bash
pnpm browse:games -i --tag=vcf-forbidden --matchup=hard --moves=20-40 --file=analyzed-games/analysis-2026-02-14T12-23-56-039Z.json --game=164
```
